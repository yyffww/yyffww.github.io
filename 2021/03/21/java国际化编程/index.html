java国际化编程

发表于2021-03-21|更新于2021-03-22|Java
|阅读量:
简介

国际化英文是 Internationalization 因为英文太长，所以缩写为il8n.

简单来说，我们要做一个多语言版本的软件。

Java是第一个设计成支持国际化的编程语言。

包

这儿我只介绍两个包

java.util.ResourceBundle 用于加载一个语言_国家语言包

java.util.Local 定义一个语言_国家

代码实现

以上便是实现国际化的一个代码，接下来我们讲解资源文件。这儿使用的是Maven项目，所以我们在资源文件夹中放入文件。命名规则是

  • 包名 + 语言 + 国家地区.properties
  • message.properties
  • message_zh.properties
  • message_zh_CN.properties
  • message_en.properties
  • message_en_US.properties

ResourceBundle会根据key 来寻找value 如果现在我们要找en_US他的寻找顺序为

message_en_US.properties

message_en.properties

message_zh_CN.properties (默认配置文件)

message_zh.properties

message.properties

意思为当没有en_US文件时，他会去找en,连en文件都没有的话，他就会读取默认的_zh_CN.properties文件,按照这个顺序，最终读到.properties文件。

语言文件(资源文件)

存储文件必须是ASCII码文件

如果是ASCII以外的文字，必须用Unicode的表示\uxxx

使用native2ascii.exe 进行转码。（我用的jdk11,好像已经取消了这个,不过可以在资源文件中敲中文，自动转码）
