图(BFS&DFS)

什么是图

图是由顶点的集合和边的集合组成的。

说到图我们通常会想到地图。地图一般会标有城市名称，以及链接各个城市的线路。在数据结构中我们称这些城市为顶点（Vertxe,也被叫做结点），而连接这些城市的线路我们称为边（Edge）。也就是说明由顶点和边组成的图形就称为图。

有向图：

权：在实际应用中图的边往往需要表示成为某种数值，这个数值便是该边的权。无向图带权就是无向带权图，有向图加上权值就是有向带权图。

BFS 与 DFS
BFS(Breadth First Search): 广度优先搜索

任意选择一个点做根结点，然后找与根结点连接的顶点，如上图我们选择A作为根结点，接下来找到B.C,又根据B或者C来找他的邻接点(这儿以B开始找)，B的邻接点是A,C,D不过A,C已经被找过了，所以只能找到D,C的邻接点是A,B,D,E 由于A,B,D也被找过了，所以也只能找到E。现在找与D相邻的结点(发问？为什么不从E找，要从D找………当然我们是按照顺序找的啦~)，D相邻的没有找过的点只有E。所以我们找出的一种结果为A，B，C，D，E，F。这儿可能解释不怎么清楚。这儿用下面一张图解释。

DFS(Deep First Search): 深度优先搜索

我的理解是 ：一个人走一条路走到黑，原路返回到上一个路口，如果这个路口有其他路，又顺着这条路走到尽头，但是除了原路返回不会到其他已经到过的地方。直到回到最初的起点。

还是拿上面图举个栗子，从A出发，顺着一条路走，我这还是走B,然后D,F。当走到F没有路时，此时原路返回到D,此时发现有两条路可走，随便走一条走到尽头。我这从E走，然后到C。现在发现A,B,D都已经到过，原路返回到E,没路，又原路返回到D,以此类推。最终回到A。

所以走过的路径依次为A，B，D，F，E，C。
