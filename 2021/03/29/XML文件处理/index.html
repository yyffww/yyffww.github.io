XML文件处理

发表于2021-03-29|更新于2021-04-02|JavaXML
|阅读量:
1.简介

XML 指可扩展标记语言（EXtensible Markup Language）。

数据是主体，但是单独的数据含义很模糊，数据+含义适用于传输数据，而不是显示数据（HTML）。

2.常规语法
  • 起始标签对应结束标签
  • 简写 可写为类似HTML
  • 大小写敏感
  • 每个文件需要一个根元素
  • 按顺序嵌套，不可错位
  • 所有属性必须有值，用双引号包围
  • 转义字符，如 “<” 用&lt；代替
  • 注释与HTML一样。

转义字符

&lt; < 小于
&gt; > 大于
&amp; & 和号
&apos; ' 单引号
&quot; " 双引号
3.拓展
  1. DTD(Document Type Definition) 定义文档结构
  2. XML Schema(XSD,XML Schema Definition) 还是定义XML 文档结构。DTD继任者。spring框架中使用的就是他.
  3. XSL (eXtensible Stylesheet Language) 扩展样式表语言。作用相当于HTML中的CSS
4.xml解析

— 树结构

  • DOM:Document Object Model 文档对象模型，擅长(小规模)读/写。

    – 解析大数据量的XML，会遇到内存泄漏及程序崩溃的风险。

— 流结构

  • SAX: Simple API for XML 流机制解释器(推模式)，擅长读。
  • Stax: The Streaming API for XML,流机制解释器(拉模式),擅长读，jdk6引入。
1).xml解析（DOM）

xml写入（DOM）

2).xml解析 (SAX)

采用事件/流模型来解析xml，更快速，更轻量，有选择的解析和访问，内存要求较低。

SAX对XML的解析为一次性读取，不创建/不存储文档对象，很难同时访问文档中的多处数据。

推模型。每发现一个节点引发一个事件，需要编写这些事件的处理程序。关键类：DefaultHandler

3).xml解析 (Stax)

Streaming API for XML

  • 采用流模型中的拉模型
  • 遍历文档时，会选择感兴趣的部分并拉出来，不需要引发事件，允许我们选择性地处理节点。
  • 两套处理API （基于指针的API XMLStreamReader）(基于迭代器的API XMLEventReader)
