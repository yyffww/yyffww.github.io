2.建立Web项目

与上述不一样的是第2步。我们不勾选跳过。选择maven-archetype-webapp

输入你的坐标,版本这些。

点击完成，不看不知道，一看吓一跳，目录不对，jsp也报错。jsp报错的原因，是应该没有服务器Tomcat.

在项目右键，点击properties。找到java修改成对应的版本，勾选Dynamic Web Module,勾选上你的tomcat。

然后试着运行下这个jsp页面。现在我们来解决项目结构问题。项目右键Build Path –>Configure Build Path

上面已经写着这两个文件夹missing。还差一个测试资源文件夹，我们添加它。

如下图，点击Contains test sources,我们让他变成测试文件夹，让它变绿。创建就基本成功。

同时这儿也设置我们对应的jdk，以免不必要的错误。

然后ok。现在你可以像IDEA一样玩耍了Web项目了。

5.maven讲解

1.依赖配置与依赖传递
1).依赖传递(依赖具有传递性)
  • 直接依赖：在当前项目中通过直接依赖。
  • 间接依赖：项目A依赖另外一个项目B，项目B依赖项目C,则C的资源能被A使用。
2).依赖冲突问题
  • 路径优先：依赖中出现相同资源时，层级越深，优先级越低
  • 声明优先：相同层级中，配置顺序靠前的覆盖靠后的
  • 特殊优先：同级配置中，相同资源不同版本，后配置的覆盖先配置的。
