Java并发数据结构&并发协作

发表于2021-04-29|更新于2021-08-31|Java
|阅读量:
1.简介

  • 我们常用的数据结构在并发编程中是不安全的。如ArrayList,HashMap.HashSet。

  • Vector,Hashtable同步集合性能过差。

  • 并发数据结构：数据添加和删除

    阻塞式集合：当集合为空或者满的时候，等待

    非阻塞式集合：当集合为空或者满的时候，不等待，返回null或者异常。

2.并发数据结构
1.List
  • Vector 同步安全，写多读少
  • ArrayList 不安全
  • Collections.synchronizedList(List list) 基于synchronized,效率差。
  • CopyOnWriteArrayList 读多写少，基于复制机制，非阻塞。
2.Set
  • HashSet 不安全
  • Collections.synchronizedSet(Set set) 基于synchronized,效率差。
  • CopyOnWriteArraySet 读多写少，基于CopyOnWriteArrayList实现，非阻塞。
3.Map
  • Hashtable 同步安全，写多读少
  • HashMap 不安全
  • Collections.synchronizedMap(Map map) 基于synchronized,效率差。
  • CopyOnWriteArraySet 读多写少,非阻塞。
4.Queue&Deque
  • ConcurrentLinkedQueue 非阻塞
  • ArrayBlockingQueue / LinkedBlockingQueue 阻塞
3.并发协作
1.synchronized 同步

限定只有一个线程才能进入关键区。

简单粗暴,性能损失大。

Monitor概念:

对象头格式

  • Monitor被翻译为监视器或者管程。

    每个Java对象都可以关联一个monitor对象，如果使用synchronizd给对象上锁之后，该对象的Mark Word 中就被设置指向Monitor对象的指针。

偏向锁:

  • 如果开启了偏向锁(默认开启),对象创建后,markword值为0x05即后3位为101,这时它的thread,epoch,age,都为0.

  • 偏向锁默认是延迟的,不会在程序启动时立即生效,如果想避免延迟,可以加VM参数XX:BiasedLockingStartupDelay=0来禁用延迟。

  • 如果没有开启偏向锁，对象创建后，markword值为0x01即后三位为001，这时它的hashcode，age，都为0，第一次使用hashcode才会赋值。

3.Semaphore

  • 信号量:本质是一个计数器.

  • 计数器大于0,可以使用,等于0不能使用.

  • 可以设置多个并发量,例如限制10个访问.

  • Semaphore

    1. acquire 获取
    2. releasc 释放

  • 比Lock更进一步,可以控制多个同时访问关键区

代码示例

3.Latch

  • 等待锁,是一个同步辅助类。

  • 用来同步执行任务的一个或者多个线程。

  • 不是用来保护临界区或者共享资源

  • CountDownLatch 向下计数类

    countDown() 计数减一

    await() 等待latch变成0

6.Barrier

  • 集合点,也是一个同步辅助类。

  • 允许多个线程在某一个点上进行同步

  • CycliBarrier

    构造函数是需要同步的线程数量。

    await等待其他线程，到达数量后，就放行。

当在Barrier上await的线程数量达到预定的要求后,所有的await的线程不再等待,全部解锁.并且,Barrier将执行预定的回调动作。

7.Phaser

  • 允许执行并发多个阶段任务,同步辅助类

  • 在每一个阶段结束的位置对线程进行同步,当所有的线程都达到这一步,再进行下一步

  • Phaser

    arrive() 到达不等待

    arriveAndAwaitAdvance() 到达等待

8.Exchanger

  • 允许在并发线程中互相交换消息

  • 允许在2个线程中定义同步点,当两个线程都到达同步点,他们交换数据结构

  • Exchanger

    exchange() ,线程双方互相交换数据

    交换数据是双向的

  • 无法做到点对点发送消息.

  • 当两个线程都同时执行到同一个exchanger的exchange方法,两个线程就相互交换数据,交换是双向的.
