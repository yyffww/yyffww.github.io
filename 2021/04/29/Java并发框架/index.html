Java并发框架

发表于2021-04-29|更新于2021-05-16|Java
|阅读量:
1.简介
1.为什么需要并发框架?

肯定是因为数据多,数据量大。

2.串行编程与并行编程。

  • 串行编程简单，拥有顺序结构，选择结构，循环结构。

  • 并行编程困难，主要是任务分配和执行过程高度耦合。

    –如何控制粒度，切割任务。

    –如何分配任务给线程，监督线程执行过程。

3.并行模式

1.主从模式(Master-Slave)

2.Worker模式(Worker-Worker) 也叫P2P

4.Java并发编程
  • Thread/Runnable/Thread组管理
  • Executor
  • Fork-Join框架
2.线程组管理
线程组ThreadGroup

树形结构

通过enumerate遍历组内线程

能够有效管理多个线程，但是管理效率低

任务分配和执行过程高度耦合

重复创建线程，关闭线程操作，无法重用线程。

方法 意义
1 ThreadGroup.activeCount() 返回线程组中还处于active的线程数
2 ThreadGroup.enumerate() 将线程组中active的线程拷贝到数组中
3 ThreadGroup.interrupt() 对线程组中所有的线程发出interrupt信号
4 ThreadGroup.list() 打印线程组中所有的线程信息
3.Executor
  • 分离任务的创建和执行者的创建
  • 线程的重复利用
1.理解共享线程池的概念

预设好的多个Thread,可弹性增加

多次执行很多很少的任务

任务创建和执行过程解耦

程序员无需关心线程池执行任务过程

2.主要类

Executors.newCachedThreadPool / newFixedThreadPool 创建线程池

ExecutorService 线程池服务

Callable 具体的逻辑对象 与Runnable等价的，可以用来执行一个任务，可以有返回值。

Future 返回结果

4.Fork-join
  • java 7开始提供的另外一种并发框架:分解,治理,合并….(和学过的归并排序一样的逻辑)
  • 适用于整体任务量不好确定的场合。
