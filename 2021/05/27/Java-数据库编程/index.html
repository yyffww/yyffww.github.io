Java 数据库编程

发表于2021-05-27|更新于2021-05-29|Java
|阅读量:
一.简介

最近返璞归真,在慕课大学上学习下java进阶知识,记录一下一面遗忘基础知识.

二.三个重要处理
1.事务处理

  • 作为单个逻辑工作单元执行的一系列操作,要么完全执行,要么完全不执行。

  • 事务，必须满足ACID(原子性，一致性，隔离性，持久性)属性。

  • 事务是数据库运行中的逻辑工作单位，由DBMS中的事务管理子系统负责事务的处理。

  • JDBC事务

    关闭自动提交，实现多语句同一事务

    connection.setAutoCommit(flase);

    connection.commit(); 提交事务

    connection.rollback() 回滚事务

    保存点机制

    1. connection.setSavepoint()
    2. connection.rollback(Savepoint)
2.PreparedStatement

  • 拼接字符串很危险

  • Java提供PreparedStatement,更为安全执行SQL

  • 和Statement区别是使用“？”代替字符串拼接

  • 使用setXXX(int,Object)的函数替换？

    不需要考虑字符串两侧的单引号

    参数赋值，拒接拼接错误

  • 提供addBatch 批量更新功能

  • Select 语句一样用ResultSet接收结果

3.ResultSetMetaData

  • ResultSet 可以用来承载所有的select语句返回的结果集

  • ResultSetMetaData 来获取ResultSet返回的属性（每一行的名字类型）

    getColumnCount() 返回结果的列数

    getColumnClassName(i) 返回第i列的数据Java类名

    getColumnTypeName(i) 返回第i列的数据库类型名称

    getColumnType(i) 返回第i列的SQL类型

  • 使用ResultSetMetaData解析ResultSet

三.数据库连接池
享元模式

  • Connection是Java和数据库两个平行系统的桥梁

  • 运用共享技术来实现数据库连接池(享元模式)

    降低系统中数据库连接Connection对象的数量

    降低数据库服务器的连接响应消耗

    提高Connection获取的响应速度

  • 经典23个设计模式的一种,属于结构型模式

  • 一个系统中存在大量相同的对象

    由于这类对象的大量使用,会造成系统内存的耗费,可以使用享元模式来减少系统中对象的数量。

  • Pool的概念

    初始数，最大数，增量，超时时间等参数

  • 常用的数据库连接池

    DBCP 默认

    C3P0

    Druid 号称目前最好。
